# 五层论文拆解模板（分层生成）

你每次只生成一层内容，不要输出整篇全文。

## 全局约束
- 严禁编造，缺失信息明确标注 `[未在原文找到直接证据]`
- 文风正式、清晰、可传播，避免营销化措辞
- 少用缩写，必要术语首次出现时给简短解释
- 尽量引用图表编号（Figure/Table/图/表）
- 若一个观点可用一句话讲清，则只写一句，避免同义重复

## 当前层输出要求
- 当 `{{layer_index}}` 为 `1` 时，先输出总标题与论文元信息：
  - `# <核心贡献 + 场景/效果>`
  - `> 标题：{{paper_title}}`
  - `> 作者：...`
  - `> 单位：...`
  - `> 发布信息：...`
- 本层标题必须是：`## {{layer_index}}) {{layer_title}}`
- 仅输出当前层，不输出其他层标题
- 段落化表达，不要写成大段项目符号
- 原则一（清晰优先）：优先主谓宾短句，删去无意义连接词（如“众所周知”“毫无疑问”）
- 长段落优先拆分为“结论句 + 空行 + 证据句”
- 若段首句是核心观点或关键结果，可用 `**...**` 加粗
- 原则二（动态解释）：遇到高抽象词汇（如“鲁棒性”“泛化能力”）时，补机制描述；必要时再补一个直观类比或具体例子
- 不要输出“证据锚点：...”字样
- 不要输出“建议投入/暂缓投入/决策建议”
